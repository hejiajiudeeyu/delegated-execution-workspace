# 双语文档规范

> 英文版：[./bilingual-convention.md](./bilingual-convention.md)
> 说明：中文文档为准。

本文定义本仓库双语文档的维护方式。

## 适用范围

- 适用于 Wave1 的根目录治理文档以及 `docs/` 下全部目标文件。
- 英文/原始文件保持为 `*.md`。
- 中文配对文件使用同目录下的 `*.zh-CN.md`。

## 配对规则

1. 原始文件路径与文件名不变。
2. 每个目标 Markdown 文件都必须有一个中文配对文件。
3. 中英文应保持一致的标题层级和关键步骤。
4. 若双语内容有冲突，以中文文档为准。

## 中文文件头格式

每个中文文件必须以以下四行开头：

1. `# <中文标题>`
2. 空行
3. `> 英文版：<relative-link-to-original-md>`
4. `> 说明：中文文档为准。`

## 翻译原则

- 中短文档优先完整翻译。
- 长文档可采用“精简但完整”的中文改写，必须保留：
  - 核心约束
  - 必要校验项
  - 命令与操作步骤
  - 所有权与边界说明
- 命令块通常保持原始 Shell 形式。

## 维护流程

1. 先更新英文/原始 `*.md`。
2. 同次变更内同步更新配对 `*.zh-CN.md`。
3. 校验每个目标 `*.md` 都有 `*.zh-CN.md` 对应文件。
4. 如涉及 CI 关键引用（例如 `nx.json` 的共享路径），同步更新并验证。
